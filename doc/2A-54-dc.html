2A-54-dc: Stratiform/Convective Map 
Last updated: 7/31/2003

Synopsis

2A-54-dc [-v] [options_list] qc_ghdf_file 2A-55_ghdf_file csmap_ghdf_file
singlevos_2A-54-dc [-v] [options_list] 2A-55_shdf_file csmap_shdf_file
where:

options_list := [-S site_name] [-d YYMMDD] [-H horizontal_resolution] 
      [-l result_log_file] [-D output_dir] [-m metadata_fname]

Description

This is the Dual CAPPI version of the radar site convective/stratiform mapping program. 2A-54-dc uses the same 2A-54 algorithm, but instead of using the 1C-51 base scan as input, it uses the 1.5 and 3.0 km constant altitude reflectivity planes (CAPPIs) from 2A-55 granular HDF files. 2A-54-dc calls singlevos_2A-54-dc for each VOS in the granular product. The algorithm determines whether a given pixel is associated with convective rain, stratiform rain, no rain, or missing/bad data, for each point in the scan and produces a convective/stratiform map. 2A-54-dc produces csmaps for a one hour-granule.

Options

2A-54-dc and singlevos_2A-54-dc recognize the following options:

-v Prints arguments and status of the program execution.

-S Specify TSDIS site name. It is case insensitive. See file, $GVS_DATA_PATH/gv_radar_site_info.data, for a list of sites. Default: Will obtain from the UF file.

-d YYMMDD: Date of radar data. Default is the date string from 1C-51_product (*<YYMMDD>*).

-H Specify horizontal resolution in km (2.0 or 4.0). Default: 2.0.

-l result_log_filename: Log file produced by 2A54 (default = "logfile").

-D output_dir: Directory for the result file(s) (default = "./"). Note: If 2A-54_product name includes path, then output_dir will not be used.

-m Specify filename for metadata file. Default: $GVS_DATA_PATH/2A54_metadata.txt.

Input Files

  1. qc_ghdf_file - Quality controlled reflectivity for one hour-granule in Hierarchical Data Format (HDF) (1C-51 products). It must follow the following naming convention in order to use default date value: *<yymmdd>*. This file is now used only for metadata retrieval.
  2. 2A-55_ghdf_file - Three Dimensional Reflectivity for one hour-granule in Hierarchical Data Format (HDF).
  3. 2A-55_shdf_file - Three Dimensional Reflectivity for single VOS in Hierarchical Data Format (HDF) (2A-55 product).
  4. $GVS_DATA_PATH/gv_radar_site_info.data -This is not specified on the command line (Note: Check and modify the DARW entry if necessary when data is from Darwin).

Temporary Files

The following files are created and removed upon completion.
    2A-54-dc_<process_id>/

Output Files

  1. csmap_shdf_file - Convective/stratiform map for a single VOS in HDF file. Map is 151x151 cartesian grid with the radar located at center of grid box at location (76,76). The entries are unitless as followed: 0 -> no echo; 1 -> stratiform; 2 -> convective; -99 -> missing/bad data.  No echo means the pixel from the basescan has missing value. Missing/bad means the distance of the pixel is less than 13 or greater 150.
  2. logfile (if '-l' was specified).
  3. csmap_ghdf_file - Convective/stratiform map for one hour-granule in HDF file.

Return value

2A-54-dc returns 0 upon successful completion; 1 upon successful processing for some VOSs of granule; non-zero, otherwise.

singlevos_2A-54-dc returns 0 upon successful completion; -2 upon interruption; -1 upon failure.
