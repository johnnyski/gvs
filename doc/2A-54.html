2A-54: Stratiform/Convective Map 
Last updated: 8/12/98

Synopsis

2A-54 [-v] [options_list] qc_ghdf_file csmap_ghdf_file

singlevos_2A-54 [-v] [options_list] qc_uf_file csmap_shdf_file
where:

options_list := [-S site_name] [-d YYMMDD] [-H horizontal_resolution] [-s sprint_input_template] 
      [-l result_log_file] [-D output_dir] [-m metadata_fname]

Description

Radar site convective/stratiform map. 2A-54 calls singlevos_2A-54 which interpolates the base scan for a single VOS of 1C-51 product from polar to cartesian coordinate. The algorithm determines whether a given pixel is associated with convective rain, stratiform rain, no rain, or missing/bad data, for each point in the interpolated base scan and produces a convective/stratiform map. 2A-54 produces csmaps for one hour-granule.

Options

2A-54 and singlevos_2A-54 recognize the following options:

-v Prints arguments and status of the program execution.

-S Specify TSDIS site name. It is case insensitive. See file, $GVS_DATA_PATH/gv_radar_site_info.data, for a list of sites. Default: Will obtain from the UF file.

-d YYMMDD: Date of radar data. Default is the date string from 1C-51_product (*<YYMMDD>*).

-H Specify horizontal resolution in km (2.0 or 4.0). Default: 2.0.

-s sprint_input_template: File contains SPRINT commands. Default is set to '$GVS_DATA_PATH/sprint_base_scan.inp' or '/usr/local/trmm/data/sprint_base_scan.inp' when $GVS_DATA_PATH = ""). See option -s of uf2mudras for detail on creating your own template.

-l result_log_filename: Log file produced by 2A54 (default = "logfile").

-D output_dir: Directory for the result file(s) (default = "./"). Note: If 2A-54_product name includes path, then output_dir will not be used.

-m Specify filename for metadata file. Default: $GVS_DATA_PATH/2A54_metadata.txt.

Input Files

  1. qc_uf_file - Quality controlled reflectivity for a single VOS in Universal Format (1C-51 product). It must follows the following naming convention in order to use default date value: *<yymmdd>*.
  2. qc_ghdf_file - Quality controlled reflectivity for one hour-granule in Hierarchical Data Format (HDF) (1C-51 products). It must follows the following naming convention in order to use default date value: *<yymmdd>*.
  3. sprint_base_scan.inp (default for sprint_input_template).
  4. $GVS_DATA_PATH/gv_radar_site_info.data -This is not specified on the command line (Note: Check and modify the DARW entry if necessary when data is from Darwin).

Temporary Files

The following files are created and removed upon completion.
    2A-54_<process_id>/

Output Files

  1. csmap_shdf_file - Convective/stratiform map for a single VOS in HDF file. Map is 151x151 cartesian grid with the radar located at center of grid box at location (76,76). The entries are unitless as followed: 0 -> no echo; 1 -> stratiform; 2 -> convective; -99 -> missing/bad data.  No echo means the pixel from the basescan has missing value. Missing/bad means the distance of the pixel is less than 13 or greater 150.
  2. logfile (if '-l' was specified).
  3. csmap_ghdf_file - Convective/stratiform map for one hour-granule in HDF file.

Return value

2A-54 returns 0 upon successful completion; 1 upon successful processing for some VOSs of granule; non-zero, otherwise.

singlevos_2A-54 returns 0 upon successful completion; -2 upon interruption; -1 upon failure.
