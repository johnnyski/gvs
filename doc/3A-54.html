3A-54: Monthly Site Rainfall Map

Synopsis

3A-54  [-v] [-m metadata_file]
      2A-53_HDF_infile HDF_outfile vos_times_ASCII_outfile
      inter_acc_map_ASCII_InOut inter_admin_info_file_ASCII_InOut 
      {BEGIN | MIDDLE | END} year month max_time_gap const_time_gap

Description

3A-54 product is a map of monthly surface rain totals derived from the instantaneous rain rate maps (2A-53). The map is in Cartesian coordinates with a 2 km horizontal resolution and covers an area of 300 km x 300 km at single radar sites. This monthly rainfall map is not a simple accumulation of the instantaneous maps as gaps in the data must be factored into the calculation. The rainfall ranges from 0.0 to 10,000.0 mm. It is multiplied by 10 and stored as a 2-byte integer.

Inputs

  • 2A-53_HDF_infile: Granule rainfall map (2A-53) Hierarchical Data Format (HDF) file. Compression is allowed.
  • inter_acc_map_ASCII_InOut: Intermediate accumulated rain map file in ASCII format. If the BEGIN flag is specified, this file will be created; if either the MIDDLE or END flag is specified, this file will be used in the accumulation calculation and will be updated with the new result.
  • inter_admin_info_file_ASCII_InOut: Administrative information file in ASCII format. It contains the information required to build product. If the BEGIN flag is specified, this file will be created; if either the MIDDLE or END flag is specified, this file will be used in the accumulation calculation and will be updated.
  • BEGIN|MIDDLE|END: Flag indicates whether this is the beginning, middle, or end of the monthly period. It indicates whether to create the product in HDF format and/or to store the intermediate product in ascii files.

    • If the specified flag is END, it will create the HDF file and the ascii file that contains the dates and times of all VOSes included in the calculation. Note: If either flag BEGIN or MIDDLE is specified, no HDF file nor vos_times_ASCII_oufile will be created.

    If the specified flag is BEGIN, it will create the two intermediate ascii files: one contained the time of the last VOS and the list of times for all VOSes included in the accumulation, the second file contained the accumulated rainfall map.

    If the specified flag if either MIDDLE or END, the two intermediate ascii files will be modified.

  • year: Four digit year.
  • month: The number of the current month being processed. Number starts at 1.
  • max_time_gap: Maximum time gap (minutes) between two consecutive VOSs for which VOS time difference factor will be used.
  • const_time_gap: Constant time factor (minutes) to use if time gap between two VOSs exceeds max_time_gap.

Temporary File or Directory

    3A-54_<process_id>/

Output Files

  • HDF_outfile: Monthly site rainfall map (3A-54 product) in HDF format. This file will only be created when the END flag is specified. When creating the HDF product, it will use some of the metadata values from the input granule HDF file (2A-53) and the metadata file (see option -m).
  • vos_times_ASCII_outfile: File contained a list of dates and times of all VOSes included in the product. This file will only be created when the END flag is specified.
  • inter_acc_map_ASCII_InOut: Intermediate accumulated rain map file in ASCII format. See its entry in the Inputs section.
  • inter_admin_info_file_ASCII_InOut: Administrative information file in ASCII format. See its entry in the Inputs section.

Options

3A-54 recognizes the following options:

-v: Print diagnostic messages.

-m: Specify filename for metadata file that contains some of program specific metadata values. Default:$GVS_DATA_PATH/3A54_metadata.txt


Return value

3A-54 returns 0 for successful completion; -1 for failure; or -2 for interruption.
