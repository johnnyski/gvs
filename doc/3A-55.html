3A-55: Monthly 3-D Structure

Synopsis

3A-55  [-v] [-m metadata_file]
      2A-55_HDF_infile HDF_outfile vos_times_outfile
      inter_acc_CFAD_InOut inter_acc_vprofile_InOut 
      {BEGIN | MIDDLE | END} year month

Description

3A-55 product provides radar site monthly 3-D structure information, including CFAD data and vertical profiles, obtained from 2A-55. The vertical profiles and CFAD data are computed at each analysis level for a month of radar data for the following categories:

  1. Total
  2. Total over land
  3. Total over sea
  4. Convective
  5. Convective over land
  6. Convective over sea
  7. Stratiform
  8. Stratiform over land
  9. Stratiform over sea
  10. "anvil" (Anvil is defined as echo aloft with no contribution to surface rain.)
  11. "anvil" over land
  12. "anvil" over sea

The vertical profiles have values ranged from -15.00 to 70.00 dBZ multiplied by 100; the values are stored as 2-byte integers.

The CFAD data are the number of pixels counted in specified height-reflectivity bin pairs for the categories listed above. Values range from 0 to 68,403,000 (3,000 radar volumes/month x 151 x 151) for single radar sites.

Inputs

  • 2A-55_HDF_infile: Granule 3-D Reflectivity (2A-55) Hierarchical Data Format (HDF) file. Compression is allowed.
  • vos_times_InOut: File contained dates and times of all VOSes.
  • inter_acc_CFAD_InOut: Intermediate accumulated CFAD file in ASCII format. If the BEGIN flag is specified, this file will be created; if either the MIDDLE or END flag is specified, this file will be used in the accumulation calculation and will be updated with the new result.
  • inter_acc_vprofile_InOut: Intermediate accumulated vertical profiles file in binary format. If the BEGIN flag is specified, this file will be created; if either the MIDDLE or END flag is specified, this file will be used in the accumulation calculation and will be updated with the new result.
  • BEGIN|MIDDLE|END: Flag indicates whether this is the beginning, middle, or end of the monthly period. It indicates whether to create the product in HDF format and/or to store the intermediate product in the intermediate files.

    • If the specified flag is END, it will create the HDF file. Note: If either flag BEGIN or MIDDLE is specified, no HDF file will be created.

    If the specified flag is BEGIN, it will create the three intermediate files: one contains the list of times for all VOSes included in the accumulation, the second file contains the accumulated CFAD data, and the third file contains the accumulated vertical profile.

    If the specified flag if either MIDDLE or END, the three intermediate files will be modified.

  • year: Four digit year.
  • month: The number of the current month being processed. Number starts at 1.

Temporary File or Directory

    3A-55_<process_id>/

Output Files

  • HDF_outfile: Monthly 3-D structure (3A-55 product) in HDF format. This file will only be created when the END flag is specified. When creating the HDF product, it will use some of the metadata values from the input granule HDF file (2A-55) and the metadata file (see option -m).
  • vos_times_outfile: File contained a list of dates and times of all VOSes included in the product.
  • inter_acc_CFAD_InOut: Intermediate accumulated CFAD data file in ASCII format. See its entry in the Inputs section.
  • inter_acc_vprofile_InOut: ntermediate accumulated vertical profiles file in binary format. See its entry in the Inputs section.

Options

3A-55 recognizes the following options:

-v: Print diagnostic messages.

-m: Specify filename for metadata file that contains some of program specific metadata values. Default:$GVS_DATA_PATH/3A55_metadata.txt


Return value

3A-55 returns 0 for successful completion; -1 for failure; or -2 for interruption.
