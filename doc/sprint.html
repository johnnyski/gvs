sprint: Interpolate polar radar data

Synopsis

sprint < sprint.inp > sprint.log

Description

Interpolates polar radar data to produce a gridded dataset, either 2D or 3D.   All runtime parameters are controlled from the input file, read from stdin.  These runtime parameters include data resolution, other input and output filenames, type of interpolation, etc.  The format of the input file will be described in further detail.  Normal operation of this program, for GVS, is usually done via the program uf2mudras.

Options

None.

Input

Standard input is read and contains runtime parameters controlling how sprint functions.  A descriptin of the format of the input file is described here.  I will only list commands used for GVS processing.
  1. Comments in this file must be 'C  ', NOTE THE TWO SPACES following the C, or '*' in column one (1), no extra spaces needed, if you use '*'.
  2. NO BLANK LINES please, but, blank comment are ok.
  3. Any charactes beyond column 80 are ignored.
  4. Each command begins in column 1 and is 8 characters wide, all uppercase.
  5. Each field, be it character or number is 8 characters wide.
Here is an example input file.  Following that, a brief description of the format.

INPUT   10                      24      21
INTERP  BI-LIN  2       1       2
C        DZ      LINEAR
        CZ      LINEAR
END
RADAR   UF                      SUNFOR
FXTABLE ACCEPT
        1
END
OUTPUT  11              BEG                     STEINER TRMM    PUR
PROCESS 981222.                         FIXED   0.5                     FXTABLE
QUIT

Return value

sprint returns 0 for successful completion; -1 for failure; -2 for interruption.
